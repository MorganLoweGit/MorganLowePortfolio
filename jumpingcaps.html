Implementation into the main game:
My role:

The class was given the task to create an arcade game that includes our own independent games as arcade machines. This project requires communication, which the team has succeeded in. This page describes my individual game.

  • Brainstorming ideas of a short term development plan, to make a small arcade game.
  • Conversed with team members to conclude on an idea.
  • Agreed on a naming convention to make organising assets / handling different peers' code easier.
  • Low poly models, based on the idea that the game should not be resource intensive.
  • Keeping the complexity of the game to a minimal, to stay on theme and to also cut on performance costs.
  • Revising scripts for ways to optimise methods.
  • Receiving feedback from game testers, that reflect the final design.
  • Planning for time to help with creating the main arcade game itself.
  • Working with peers to merge GitHub branches to form the final build.
